# ADR-003: Use Sanity as the Content Management System

**Status:** Accepted

## Context

The platform requires structured content with relationships between communities, villages, people, articles, photographs, and sources.

## Options Considered

* Sanity
* Payload CMS
* Strapi
* Custom CMS

## Decision

Use Sanity CMS.

## Rationale

Sanity provides flexible schemas, excellent editorial tools, strong support for structured content, and seamless integration with Next.js.

## Consequences

Editors can manage content efficiently while developers focus on the public-facing platform instead of building a custom CMS.
